1. Overview and EDA

Clinical Question

Can non-laboratory primary care variables identify individuals with likely diabetes?

The analysis uses NHANES 2017-March 2020 pre-pandemic cycles and includes adults aged 20 and older.

EDA ResultValueInterpretation
Initial sample size9,232Adults in the early analytic dataset.
Unweighted preliminary diabetes prevalence14.5%Overstates population burden because NHANES oversamples higher-risk groups.
Survey-weighted preliminary diabetes prevalence10.3%Better reflects the US population estimate.
Missingness was substantial and structured, so complete-case modeling was used for the baseline screening predictors.

2. Target Definition

Undiagnosed Diabetes

The project defines undiagnosed diabetes as clinically elevated HbA1c without existing diagnosis or medication treatment.

  • HbA1c >= 6.5%
  • No self-reported diabetes diagnosis
  • Not taking diabetes medication

3. Baseline Modeling

Model Setup

The baseline model is a logistic regression using simple non-laboratory predictors.

undx_dm ~ RIDAGEYR + RIAGENDR + BMXBMI + BMXWAIST + INDFMPIR

The logistic model outperformed the simple clinical rule, showing value in combining multiple predictors.

4. Feature Engineering and Model Comparison

Feature Engineering

Engineered features were designed to capture clinical risk structure that raw variables may miss.

  • Age groups such as 20-39, 40-59, and 60+
  • BMI categories such as normal/underweight, overweight, and obese
  • Waist-risk indicators using sex-specific cutoffs
  • Income-risk and age/BMI interaction indicators

Key insight: the available signal appears largely linear, so model complexity did not materially outperform logistic regression.

5. Threshold Selection

Screening Threshold

Because undiagnosed diabetes is rare, the default 0.50 threshold is not appropriate for screening. Lower thresholds were compared to prioritize sensitivity.

ThresholdSensitivitySpecificityScreening Interpretation
0.030.8590.472Selected; catches most potential undiagnosed cases.
0.050.4530.800More conservative; misses more cases.
0.070.1880.925Too conservative for screening.

A threshold of 0.03 was selected because missing true undiagnosed diabetes cases is more costly than sending additional people for confirmatory HbA1c follow-up.

6. Final Takeaways

  • Undiagnosed diabetes is a rare outcome, so PR AUC and threshold behavior matter alongside ROC AUC.
  • Logistic regression remained the best practical model because it performed well and is easy to explain.
  • Tree-based models and tuning did not materially improve performance.
  • The selected 0.03 threshold turns predicted probabilities into a screening-oriented decision rule.
  • The Shiny app intentionally uses non-laboratory inputs only; HbA1c is used for target definition and follow-up, not as a model feature.
Educational only: this report and app are screening demonstrations, not diagnostic medical tools.
